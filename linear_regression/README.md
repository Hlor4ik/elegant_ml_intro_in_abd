# Домашка 1, часть 2: Регрессия с инференсом

Здесь я делал предсказание цены подержанных машин.
Главная цель: пройти весь пайплайн от EDA до модели и потом сделать мини-сервис в Streamlit.

## Кратко, что сделано

1. Почистил и подготовил данные.
2. Обучил несколько регрессионных моделей.
3. Сравнил их по метрикам качества.
4. Добавил категориальные признаки через One-Hot Encoding.
5. Выбрал лучшую модель и сохранил ее в pickle.
6. Сделал Streamlit-приложение для предсказаний.

## Основные шаги в ноутбуке

### 1) EDA и подготовка
- Удалил дубликаты.
- Преобразовал признаки типа mileage/engine/max_power/torque в числовые.
- Разобрал torque на отдельные части.
- Заполнил пропуски.
- Посмотрел корреляции и графики.

### 2) Модели на числовых признаках
Пробовал:
- Linear Regression
- Lasso
- ElasticNet

Вывод: на одних только числовых фичах качество среднее (R2 около 0.59-0.60).

### 3) Добавил категориальные признаки
- Из name оставил бренд (первое слово).
- Сделал One-Hot Encoding для категорий + seats.
- Обучил Ridge через GridSearchCV (10 фолдов).

Итог: качество сильно выросло, лучшая модель дала R2 test около 0.78.

### 4) Бизнес-метрики
Считал 2 специальные метрики из задания:
- доля предсказаний в диапазоне +-10% от реальной цены;
- асимметричная метрика, где занижение штрафуется сильнее.

По этим метрикам лучшей тоже оказалась Ridge + OHE.

## Финальный выбор модели

В приложение отправил Ridge-модель на OHE-признаках,
потому что она показала лучшие результаты по R2 и по бизнес-метрикам.
